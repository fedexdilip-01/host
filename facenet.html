FaceNet Architecture

 ┌─────────────┐    ┌─────────────────┐    ┌──────────────┐
 │  Input      │    │   Deep CNN      │    │  Embedding   │
 │  Face Image │───►│ (Inception ResNet│───►│   L2 Norm    │
 │  160x160x3  │    │   or ZF/Incep)   │    │  128-D / 512D│
 └─────────────┘    └─────────────────┘    └───────┬──────┘
                                                    │
                                          ┌─────────▼─────────┐
                                          │ Triplet Loss      │
                                          │  L = max(d(a,p)   │
                                          │   - d(a,n) + α,0) │
                                          └───────────────────┘

Core Idea: Map face images to an Euclidean space where distance corresponds to similarity. Pre-trained model (e.g., on MS-Celeb-1M or CASIA-WebFace) fine-tuned on target dataset.

Triplet Mining: Batch Hard Embedding dim: 128 Input: 160x160 RGB MTCNN pre-alignment

Training Pipeline Overview

1. Load pretrained FaceNet (InceptionResNetV1) → remove final classification layer.
2. Add embedding layer → L2 normalize output.
3. Define triplet loss with margin α = 0.2.
4. Use hard triplet mining (semi-hard or batch hard).
5. Optimizer: Adam (lr=0.0001) with cosine decay.
6. Train for 50-200 epochs, evaluate on LFW.
Data Requirements: Aligned face crops, identity labels. Use dataset like VGGFace2, CASIA, or custom.
Evaluation: Compute accuracy on LFW (Labeled Faces in Wild) with ROC & distance threshold.
End-to-end: Pre-trained FaceNet → Fine-tuning via Triplet Loss → Feature embedding → Face verification/recognition
